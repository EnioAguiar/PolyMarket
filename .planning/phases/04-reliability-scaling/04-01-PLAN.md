Create the WebSocket client class with these features:

**Constructor options:**
- `wsUrl`: WebSocket URL (default: `wss://ws-subscriptions-clob.polymarket.com/ws/market`)
- `heartbeatInterval`: ms between PINGs (default: 10000)
- `reconnectBaseDelay`: initial reconnect delay in ms (default: 1000)
- `reconnectMaxDelay`: max reconnect delay in ms (default: 30000)

**Class: PolymarketWsClient**
- `connect(assetIds: string[])` — Connect and subscribe
- `disconnect()` — Graceful disconnect
- `subscribe(assets_ids: string[])` — Dynamic subscribe
- `unsubscribe(assets_ids: string[])` — Dynamic unsubscribe
- `isConnected()` — Connection state
- `onEvent(handler: (event: WsEvent) => void)` — Event listener
- `onConnect(handler: () => void)` — Connection listener
- `onDisconnect(handler: () => void)` — Disconnection listener
- `onError(handler: (error: Error) => void)` — Error listener

**Heartbeat implementation:**
```typescript
private startHeartbeat(): void {
  this.heartbeatTimer = setInterval(() => {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send('PING');
    }
  }, this.heartbeatInterval);
}
```

**Reconnection with exponential backoff:**
```typescript
private scheduleReconnect(assetIds: string[]): void {
  const delay = Math.min(
    this.reconnectBaseDelay * Math.pow(2, this.reconnectAttempt),
    this.reconnectMaxDelay
  );
  this.reconnectAttempt++;
  setTimeout(() => this.connect(assetIds), delay);
}
```

**Connection lifecycle:**
1. On `open` → send subscription immediately → start heartbeat → call onConnect
2. On `message` → parse JSON → call onEvent handler
3. On `close` → stop heartbeat → call onDisconnect → scheduleReconnect
4. On `error` → call onError

**Subscription message format:**
```json
{
  "assets_ids": ["<token_id_1>", "<token_id_2>"],
  "type": "market",
  "custom_feature_enabled": true
}
```

**Instance tracking:**
- Track subscribed assetIds so reconnect can re-subscribe
- Increment reconnectAttempt on disconnect, reset on successful connect
- Store interval IDs for cleanup

**Class: SubscriptionManager**

```typescript
class SubscriptionManager {
  private subscribedAssets: Set<string> = new Set();
  
  getAssetIds(): string[];  // Returns array of subscribed asset IDs
  add(assets: string[]): string[];  // Returns new assets to subscribe
  remove(assets: string[]): string[];  // Returns assets to unsubscribe
  has(assetId: string): boolean;
  clear(): void;
  count(): number;
}
```

**Features:**
- Track subscribed assets in a Set for O(1) lookup
- `add()` returns only NEW assets not already subscribed (for subscribe operation)
- `remove()` returns assets that were actually subscribed (for unsubscribe operation)
- Serializes to/from JSON for logging/debugging

**Usage with PolymarketWsClient:**
```typescript
const subscriptionManager = new SubscriptionManager();
const wsClient = new PolymarketWsClient();

// On connect: subscribe to all known assets
wsClient.onConnect(() => {
  const assets = subscriptionManager.getAssetIds();
  if (assets.length > 0) {
    wsClient.subscribe(assets);
  }
});

// When new market detected
wsClient.onEvent((event) => {
  if (event.event_type === 'new_market') {
    const newAssets = subscriptionManager.add(event.assets_ids);
    if (newAssets.length > 0) {
      wsClient.subscribe(newAssets);
    }
  }
});
```

This manager keeps track of what we're subscribed to, needed for reconnection re-subscription.
